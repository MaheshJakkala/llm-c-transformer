## 3. Inference Latency

*Protocol: 10 warm-up iterations + 100 timed iterations, averaged. Single-threaded CPU.*  
*Source: `results/metrics/full_benchmark.csv` — generated by `python3 benchmark_pytorch.py`.*  
*Python backend: NumPy with OpenBLAS — the same BLAS backend PyTorch CPU uses for `nn.Linear`.*

### All 4 Backends — Same Model, Same Hardware

| Seq | PyTorch FP32 (ms) | PyTorch INT8 (ms) | C FP32 (ms) | **C INT8-AVX2 (ms)** | vs PT FP32 | vs PT INT8 |
|---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 8  | 1.984 | 2.010 | 5.367 | **0.142** | **14.0×** | **14.1×** |
| 16 | 3.159 | 3.134 | 10.919 | **0.323** | **9.8×** | **9.7×** |
| 32 | 4.384 | 5.017 | 21.457 | **1.606** | **2.7×** | **3.1×** |
| 64 | 9.183 | 8.847 | 43.225 | **6.027** | **1.5×** | **1.5×** |

> **Why PyTorch INT8 ≈ PyTorch FP32:**  
> `torch.quantization.quantize_dynamic` stores `nn.Linear` weights as `INT8` but immediately dequantizes them back to `float32` before each matmul. The compute is still FP32. This is why the "optimal" PyTorch INT8 baseline gives ≈0% latency reduction over plain FP32.

> **Why C INT8-AVX2 is faster:**  
> `_mm256_maddubs_epi16` executes 32 INT8 multiply-accumulates per SIMD instruction. Weights remain `int8_t` through accumulation (into `int32_t` registers). At seq ≤ 16, the 4× smaller weights fit entirely in L1/L2 cache — eliminating cache misses that FP32 suffers.

> **Why the gap closes at seq=32–64:**  
> Attention scores (`seq × seq`, unquantized FP32) scale quadratically and dominate latency at longer sequences.

### MatMul Micro-Benchmark — Measured by `bench.c`

*Source: `results/metrics/matmul_bench.csv`. M=N=K=256, 500 iterations averaged.*

| Backend | Time (ms) | Throughput | vs C FP32 |
|---|:---:|:---:|:---:|
| NumPy / PyTorch CPU (OpenBLAS) | 0.376 ms | 89.33 GFLOP/s | 8.2× faster |
| C FP32 naive (triple-loop) | 3.379 ms | 9.93 GFLOP/s | 1× baseline |
| **C INT8-AVX2** | **0.619 ms** | **54.22 GOPS/s** | **5.46× faster** |

## 6. Reproducing These Numbers

```bash
# Full pipeline — build, benchmark, compare, plot
./run_benchmark.sh

# Individual steps
make bench && ./bench                  # C FP32 + C INT8-AVX2 latency + throughput
python3 benchmark_pytorch.py           # PyTorch FP32 + INT8 vs C (4-backend table)
./lm                                   # LM training — loss, time, arena stats
python3 scripts/plot_results.py        # regenerate all 8 plots

# Output files
results/metrics/bench_results.csv          # C FP32 + C INT8 latency, throughput, weight memory
results/metrics/full_benchmark.csv         # all 4 backends, all sequence lengths
results/metrics/lm_training_loss.csv       # per-epoch loss and perplexity
results/metrics/matmul_bench.csv           # matmul micro-benchmark
results/metrics/lm_training_log.txt        # arena stats, total training time
```
